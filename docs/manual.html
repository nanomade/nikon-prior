Nikon-Prior Microscope Control

User Manual — Nikon L200ND • Prior ProScan III • Allied Vision Alvium 1800 U-508c

Overview

This software provides integrated control of a Nikon L200ND upright microscope with a motorised Prior Scientific XYZ stage and an Allied Vision USB3 camera. It runs as a Python/PyQt5 desktop application and is designed for optical characterisation of 2D materials, wafers, and thin films.

All windows are launched from the Launcher panel on the right edge of the screen. Windows can be freely moved, resized, and stacked; they are not docked to a fixed layout.

Tip: The Launcher stays on top of all other windows so you can always reach it.

Note: The joystick (PS3J100) communicates directly with the ProScan III controller and moves the stage outside the software. Stage sliders will resync automatically on the next application start, or by reopening Stage Controls.

Launching the Software

cd ~/nikon-prior
source ~/nikon-prior/bin/activate     # activate venv
QT_QPA_PLATFORM=xcb python main.py

The QT_QPA_PLATFORM=xcb prefix avoids a Wayland warning; the warning is harmless if omitted. A convenience script start_nikon_prior on the desktop sets this automatically.

On startup the application:

  1. Connects to the ProScan III controller (auto-detects serial port)
  2. Opens and configures the Alvium camera (falls back to a synthetic mock if no camera is found)
  3. Shows the Camera Preview and Launcher windows
Tip: If the camera shows "already in use", run pkill -f main.py or replug the USB cable, then restart.

Camera Preview

The Camera Preview window shows the live camera feed. It scales to fill whatever window size you set — the image always fits the window while preserving aspect ratio.

Double-click to centre

Double-click any point in the preview to move the stage so that point is centred in the field of view. The movement uses the current calibrated pixels-per-µm value, so accuracy depends on the magnification calibration being correct.

Focus bar

A thin coloured bar at the bottom of the preview shows the Laplacian variance focus metric:

  • Blue — very out of focus (< 5)
  • Orange — partially in focus (5–20)
  • Grey — sharp (> 20)

The numeric value is shown to the right of the bar. Use this as a real-time focus guide when adjusting the coarse or fine focus knobs.

Imaging Controls

Open via Launcher → Open Imaging Controls.

Exposure & Gain

ControlDescription
Exposure sliderLog-scale from 0.01 ms to 1000 ms. Drag or type a value (in ms) into the text box and press Enter.
GainAnalogue sensor gain, 0–100 dB. Increase only when exposure is at its maximum — gain amplifies noise as well as signal.
Auto ExposureHands exposure control to the camera's built-in auto-exposure algorithm. The slider and text box are disabled while active; unchecking reads the current exposure back from hardware.
Mag-based exposureWhen checked, switching magnification automatically applies the stored exposure for that objective from focus_presets.json. The default for 10× is 6 ms.

White Balance

ControlDescription
WB Temp (K)Sets the Red and Blue channel BalanceRatio on the camera using a colour-temperature approximation. Range 2800 K (warm/tungsten) to 7500 K (cool/daylight). Green is fixed at 1.0 by the camera. The default 5300 K gives a neutral result on this setup.
Tip: If colours look wrong after changing objectives or light intensity, adjust WB Temp to re-neutralise the grey background.

Magnification

ControlDescription
Magnification dropdownInforms all calibrated measurements (scale bar, measure tool, double-click navigation). Options: 5×, 10×, 20×, 50×, 100×. Does not move any hardware — set this to match the physical objective in use.

Calibration values (pixels per µm) are defined in ui/preview.py → calibration_table. The 10× value was measured with a stage micrometer; others are linearly scaled from it and should be verified.

Binning

Reduces camera resolution to increase frame rate and match the field of view seen at different pixel densities.

SettingResolutionFPS (approx.)Notes
1× (full)2464 × 2056~30Full sensor, software processing is heaviest
2×1232 × 1028~30Camera ROI mode; software downscale applied to maintain full FOV
4× (default)616 × 514~30Hardware binning — fastest, best SNR, full FOV, recommended for autofocus
Tip: Use 4× for navigation and autofocus; switch to 1× for high-resolution capture. All binning modes show the same physical field of view.

Scale Bar

ControlDescription
Show Scale BarOverlay a calibrated scale bar in the corner of the preview. Length is chosen automatically (100 µm at 10×, etc.).
Scale Bar ColorWhite or Black — choose whichever contrasts with your sample background.

Zoom & Crosshair

ControlDescription
Show Centre ZoomOpens a 500 × 500 px zoom window showing a 40 × 40 px crop from the centre of the frame at 12.5× magnification. A calibrated scale bar is overlaid. Double-click in this window to move the stage so that point is centred.
Zoom Under CursorThe zoom window follows the mouse pointer over the camera preview instead of being fixed at the centre. Useful for inspecting features without moving the stage.
Full-Screen Crosshair + TicksDraws a full-width/height green crosshair with distance tick marks over the image. Tick spacing is chosen automatically from the calibration.
Native Zoom (1:1)Snaps the preview window to exactly 1 display pixel per camera pixel. Useful for pixel-level inspection at full resolution.

HUD (Heads-Up Display)

Checking Show HUD overlays a semi-transparent information panel on the preview showing: current magnification, live FPS, and X / Y / Z stage positions in mm. Updated at ~3 Hz.

Flicker Averaging

Averages the last N frames before display. Set to 1 (off) by default. Use 3–4 frames to suppress 50/60 Hz flicker from fluorescent or LED ambient lighting. Higher values introduce motion blur if the stage is moving.

Saving Images from the Controls Panel

ButtonDescription
Save ViewOpens a file dialog and saves the current rendered preview (with scale bar, crosshair, and any overlays) as PNG.
Capture FrameOpens a file dialog and saves the raw camera frame (no overlays) as PNG.

For richer filename metadata, use File Save Options instead (see File Save Options).

Measure Tool

Enable via the Measure Tool checkbox. Click two points on the preview to measure the distance and angle between them.

  • Distance — shown as N +/- 1 px / M +/- δ um, where δ is the 1-pixel positional uncertainty propagated to physical units.
  • Angle — shown as θ +/- δθ deg, where δθ is computed from the 1-pixel positional uncertainty.
  • Click a third time to clear and start again.
  • When fewer than two points are placed, the zoom window follows the cursor for precise point placement.

Focus

Objective Focus Presets

Open via Launcher → Open Focus Panel.

Stores a Z position for each objective (5×, 10×, 20×, 50×, 100×) so you can instantly refocus after changing objectives. Presets are relative to the last origin set and assume the coarse focus knob has not been moved.

ControlDescription
Set [obj]Save the current Z as the focus preset for that objective.
Goto [obj]Move Z to the stored preset.
LockPrevents a preset from being overwritten accidentally.
Lock DeltaWhen enabled, updating one objective's preset automatically shifts all unlocked presets by the same delta — useful after a coarse focus adjustment that affects all objectives equally.

Presets are saved to focus_presets.json in the application directory.

Autofocus

Open via Launcher → Open Autofocus.

Two-phase focus sweep based on the derivative of a focus metric.

Phase 1 — Coarse sweep

Steps through Z at a coarse step size, tracking the derivative of the focus metric. Direction is chosen automatically (or from the last known focus position). The scan stops as soon as a zero-crossing of the derivative is detected (metric peak). If no peak is found in one direction the algorithm reverses and tries the other direction.

Phase 2 — Fine sweep

Fits a parabola to the coarse data near the peak to estimate a sub-step accurate centre, then does a fine sweep around it. Moves to the best-found Z.

ParameterDescription
MetricFocus quality measure. LaplacianVariance (default) works best for high-contrast samples. Tenengrad is more robust to uniform textures. ImageVariance is fastest. Brightness is useful for transmitted-light samples.
Step (mm)Fine step size. Smaller = more precise but slower.
Settle (ms)Wait time after each Z move before measuring. Increase if mechanical vibration is visible.
Coarse factorMultiplier applied to Step for the coarse phase (coarse step = Step × factor).
Fine factorMultiplier for the fine phase step (fine step = Step × factor).
Fine range factorHalf-width of the fine sweep as a multiple of the coarse step.
Trend NNumber of steps used for both the initial direction check and the smoothing window. Increase for noisier samples.
Frames avgNumber of frames averaged per measurement. Increase to reduce camera noise at the cost of speed.
Z range (mm)Maximum scan distance in each direction from start. Acts as a safety budget — the scan stops at the peak, not necessarily at this distance.

A live plot shows the focus metric vs Z and its derivative during the sweep. Per-magnification defaults are loaded automatically when the magnification selector changes, and can be saved via Save as default.

Tip: 4× binning is strongly recommended for autofocus. At 1× the metric computation is slower and may cause timing issues with settle delays.

Focus Map

Open via Launcher → Open Focus Map.

Builds a 2D focus surface by running autofocus at a regular grid of stage positions across the wafer. The surface is interpolated so that during a wafer scan, Z can be set from the map without running autofocus at every point.

ControlDescription
Columns / RowsGrid density (2–15 per axis). More points = more accurate surface at the cost of measurement time.
Min metricAutofocus quality threshold. Points below this are flagged for manual refocusing.
Fit degree1 = plane, 2 = quadratic, 3 = cubic. Start with 1 (flat wafer). Increase only if the residuals are systematic.
Measure Focus MapStart the automated grid scan (requires wafer boundaries to be set).
Apply Focus HereMove Z to the interpolated depth at the current XY position.
Save Map / Load MapExport and import focus surfaces as JSON for reuse across sessions.
Set Reference ZAfter loading a map from a previous session, focus manually at any reference point and click this to anchor the offsets to the current hardware position.

The heatmap display shows the surface colour-coded by Z offset (red = higher, green = lower). Statistics (peak-to-valley ΔZ, standard deviation) are shown below the map.

Sample Tools

Find Wafer Extents

Open via Launcher → Find Wafer Extents.

Automatically locates the wafer boundary by stepping the stage outward in ±X and ±Y from the origin and detecting the intensity drop where the sample ends. Results are passed directly to the Wafer Scan panel.

ControlDescription
Step sizeDistance per step during edge search (0.05–2.0 mm). Smaller = more precise edge location but slower.
Auto thresholdIf checked, the software measures intensity at centre and at X=10 mm to automatically compute the brightness threshold. Leave unchecked if the automatic detection fails (e.g. very dark or reflective sample).
Manual thresholdFrame intensity (0–255 mean brightness) below which a frame is considered "off wafer". Active when Auto threshold is unchecked. 100 is a reasonable starting point for typical samples.
Find Wafer ExtentsStart the search. A progress dialog shows each axis being scanned. The stage returns to centre after each edge is found.
Tip: Run Find Wafer Extents before starting a Wafer Scan — the detected boundaries feed directly into the scan grid.

Wafer Scan

Open via Launcher → Open Wafer Mapping.

Performs an automated serpentine raster scan over the detected wafer area, capturing images at each grid point with optional autofocus.

ControlDescription
Step X / Step YGrid spacing in mm. Use Set step = FOV to automatically calculate the step from the current calibrated field of view (no tile overlap).
Set step = FOVComputes step size = sensor width or height / (pixels per µm) / 1000, giving exactly contiguous tiles. Reduce manually for overlap.
Settling timeWait after the stage reaches each point before capturing (default 0.5 s).
Position toleranceMaximum allowed position error before capture proceeds (default 0.01 mm).
Safe zoneInset from each detected wafer edge (default 0.5 mm). The scan grid will not approach within this distance of any boundary.
Use focus map ZBefore each capture, move Z to the interpolated depth from the Focus Map. Requires a map to have been measured. Fast alternative to per-point autofocus.
Autofocus each pointRun autofocus at every grid point. Can be combined with focus map (map provides initial Z, autofocus refines).
Start Scan / StopBegin or abort the scan.
Go to CentreMove stage to (0, 0).
Open Scan FolderOpen the output directory in the file manager after scanning.

Images are saved as PNG files named img_X+X.XXX_Y+Y.YYY.png in a timestamped folder (wafer_scan_YYYYMMDD_HHMMSS/). A JSON metadata file is also written.

Analysis Tools

Layer Contrast Detection

Open via Launcher → Open Layer Contrast.

Detects and highlights monolayer, bilayer, and trilayer regions in real time by colour thresholding the live preview.

ControlDescription
Pick (per layer)Activates eyedropper mode. Click any pixel in the preview to sample its colour as the target for that layer.
Contrast fieldEnter the optical contrast (ΔI/I in %) relative to background. The target colour is computed as background × (1 + contrast/100). Typical graphene contrasts: monolayer −2 to −4%, bilayer −4 to −8%.
Auto (background)Automatically determines background colour from the histogram mode of each frame.
Overlay colourChoose the outline colour for each detected layer type.
ShowToggle overlay visibility per layer.
ToleranceColour matching radius in RGB space (1–80). Increase for noisier images or weaker contrast.

Flat-Field Correction

Open via Launcher → Flat-Field Correction.

Removes spatially non-uniform illumination (vignetting, dust on optics) by computing a per-pixel correction map from many frames captured at random stage positions.

ControlDescription
PositionsNumber of random positions to sample (10–500). Use ≥50 for accurate correction; more is better if the sample covers a large fraction of the field.
Range (mm)Stage moves randomly within ±this distance from the start position.
Collect & Build BackgroundStart collection. The stage moves to each random position, waits 50 ms, captures a frame, then returns to the start. The median across all frames is computed as the flat field.
Apply correction to previewDivide each live frame by the flat field and stretch contrast. Toggle to compare corrected and uncorrected views.
Smoothing σGaussian blur applied to the flat field before division. σ = 0 is fine for most cases; increase to 5–10 if dust spots in the flat field create ring artefacts.
Tip: Collect flat fields at the same exposure and binning setting you will use for imaging. Recollect if you change objectives or illumination.

Pixel Intensity Measurement

Open via Launcher → Pixel Intensity.

Samples a 5×5 pixel region of interest at the frame centre and logs mean intensity, central pixel intensity, and per-pixel BGR values to a CSV file (data/pixel_data.csv) together with the current stage coordinates. Use for repeatable reflectance measurements at specific stage positions.

Calibration Notes

Pixels per µm

The calibration table in ui/preview.py maps each objective to a pixels-per-µm (ppm) value. The 10× value (2.03 ppm) was measured with a stage micrometer on 2026-03-18. All other objectives are linearly scaled from this and should be verified independently.

Objectiveppm (px/µm)Source
5×1.015Implied from 10×
10×2.030Measured 2026-03-18
20×4.060Implied from 10×
50×10.150Implied from 10×
100×20.300Implied from 10×

Step sizes and axis invert flags

Per-axis step size (mm/count), invert flags, and travel limits are stored in motors/step_config.json. The nominal step size is 0.0001 mm/count for XY. Verify against a stage micrometer and update the file if measured steps deviate more than ~2%.

Z travel

The Prior PS3H122 coupled to the Nikon fine focus via the H122KON kit gives approximately 25 mm of Z travel. The slider range is set accordingly. Do not attempt to drive Z beyond the mechanical limits of the fine focus knob.

White balance

The default WB setting is 5300 K. The mapping from Kelvin to BalanceRatio values is a linear approximation and may need adjustment for different illumination sources. If you change the illuminator, re-optimise the slider and note the new preferred value.

Troubleshooting

Camera not found / "already in use"

  • Run pkill -f main.py to kill any stale process.
  • Replug the USB3 cable.
  • Check that the Vimba X runtime is installed: ~/Desktop/VimbaX_2025-1/bin/VimbaXViewer should open without error.
  • If VmbPy is not installed: pip install vmbpy in the venv after running InstallGenTL.

Stage not responding

  • Check that /dev/ttyACM0 exists: ls /dev/ttyACM*. Replug the USB cable if absent.
  • Confirm no other process has the port open: fuser /dev/ttyACM0.
  • The ProScan III controller must be powered on before the software is started.

Sliders out of sync with hardware after joystick use

The Prior PS3J100 joystick communicates directly with the ProScan III and moves the stage without notifying the software. Reopen Stage Controls or restart the application to resync.

Autofocus fails at 1× binning

Use 4× binning for autofocus. At 1×, frame computation is slower and the settle timing may be insufficient. The autofocus parameters are tuned for the 616 × 514 frame size produced by 4× binning.

Find Wafer Extents finds wrong edges

  • Adjust the manual threshold to match your sample's brightness.
  • Make sure the stage origin (0, 0) is near the centre of the wafer before starting.
  • If the background is only slightly dimmer than the sample, try increasing the step size to speed up the search and rely on the consecutive-dark-frame confirmation logic.

WB slider has no effect / gives errors

The camera must be streaming (Camera Preview open) for WB changes to take effect. Green channel is not adjustable on the Alvium (it is fixed at 1.0); only Red and Blue ratios are set.

Wayland / Qt platform warning at startup

The warning "could not connect to display" or "xcb platform plugin not found" is harmless when using QT_QPA_PLATFORM=xcb. The application will use the X11 backend regardless of the Wayland compositor.
